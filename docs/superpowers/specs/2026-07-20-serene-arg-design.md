# 《静园 / Serene》— ARG 恐怖解谜网页游戏 · 设计文档

- 日期：2026-07-20
- 类型：中篇 · 纯静态多页 · 单人可完整开发
- 一句话：一款"细思极恐"的替代现实游戏（ARG）。玩家扮演外部调查者，深挖一款伪装成温柔治愈的心理健康冥想 App「静园」，逐层揭露它采集情绪数据、"抹除负面人格"（实为记忆/意识清除）的真相；中途游戏认出玩家本人，第四面墙破裂。

---

## 1. 核心概念

- **题材内核**：伪装成正能量组织的邪教/实验。外壳是心理健康冥想自助 App「静园 / Serene」，主打"帮你摆脱负面情绪、找回真我"。深处：用户情绪数据被采集，"负面人格"被"清除"，所谓"重生"实为意识/记忆抹除。
- **玩家视角**：外部调查者。从一封"转错的邮件"里的链接进入静园官网，一步步逼近真相 —— 直到游戏识别出"你"其实也早已是它的目标/样本（A→B 视角融合，第四面墙破裂）。
- **基调**：温柔的表皮 + 越挖越扭曲的崩坏。恐怖是"熬"出来的，靠细节失真与"虚构渗入现实"的不安感，而非 jump scare。

## 2. "世界"结构（模拟多个真实网站 / 社交痕迹）

1. **静园官网**（治愈表皮，主入口）：首页、情绪测评、冥想练习、"学员故事"、关于我们、隐藏登录入口。
2. **失踪者博客**：注册用户「林」的个人博客，从满怀希望到日记崩坏，最后一篇戛然而止。
3. **假微信聊天记录**（截图 / 导出样式）：林与家人、与"静园客服"的对话，语气从热情到突变。
4. **假论坛 / 小组帖**（豆瓣小组 / 贴吧风）：一个"静园劝退 & 求助"帖，网友讨论、有人报家人失联，楼层里藏线索；越往后回帖的人越少、头像变灰。
5. **假朋友圈 / 微博动态**：林的日常打卡逐渐变味，最后一条定格。
6. **员工 / 管理后台**（需解谜取得密码）：数据库片段、"样本"编号、被涂黑的知情同意书、"重生进度"面板。
7. **"系统"层**：404 页、robots.txt、页面源码注释、浏览器控制台留言 —— ARG 味的隐藏线索。

## 3. 叙事弧线与结局

- **第一幕（温柔假象 · 崩坏档 1）**：正常浏览官网，发现细节不对劲（林的痕迹、异常数字"47"）。
- **第二幕（裂缝 · 崩坏档 2）**：解谜进入后台，拼出"抹除计划"，界面开始认出你、假装读取你的数据。
- **第三幕（重击 · 崩坏档 3）**：高潮 —— 直呼玩家、凝视、真相揭晓。
- **多结局**：
  - ① 「登出」逃离（较好结局）。
  - ② 「接受重生」（坏结局）。
  - ③ 硬核隐藏结局：挖出被埋的高阶线索，发现调查者本人才是"第 48 号样本"，揭示循环（"你已经开始过 47 次了"）。

## 4. 谜题设计（亲民探索向 · 分层门禁）

- **主线谜题**（人人可解，保证通关）：
  - 查看网页源代码找隐藏链接 / 页面。
  - 留意异常日期、编号、数字（如反复出现的 47/48）。
  - 简单替换密码：凯撒、摩斯。
  - 图片里的隐藏文字 / 明显的视觉线索。
  - 拼凑博客 + 聊天 + 论坛的时间线，推出进入后台的密码。
  - 每解开一环 → 解锁下一站点。
- **硬核彩蛋**（可选，通向隐藏结局）：藏在 HTML 注释、robots.txt、图片隐写里的高阶线索。
- **进度记录**：`localStorage` 保存已解锁站点、腐化度、结局状态。

## 5. 崩坏系统

- 全局"腐化度"（corruption level）变量随主线进度提升。
- 由 CSS / JS 驱动页面从档 1 → 档 2 → 档 3 递进扭曲：
  - **档 1 温柔渗血**：文案里悄悄多出不该有的句子、诡异计数。
  - **档 2 界面撕裂**：RGB 撕裂闪烁、界面"认出你"、假装调用摄像头 / 读取数据。
  - **档 3 生理恐怖**：凝视的眼睛、扭曲的脸、直呼玩家、暗红脉动（留给高潮 / 结局的一击）。

## 6. 技术架构

- **纯静态多页**：原生 HTML + CSS + JavaScript，无框架（更贴合"模仿一个个真实老网站"的质感）。
- **状态**：`localStorage` 存进度、腐化度、结局。
- **必要时**：个别需要"服务端保密"的谜题，可加极轻量 serverless 函数（起步阶段尽量不用）。
- **部署**：GitHub Pages / Netlify（免费、静态）。
- **目录**：按"多个网站"组织（如 `/serene/`、`/blog/`、`/forum/`、`/admin/` 等）。
- **答案保护**：主线关键答案做编码 / 混淆，避免直接读源码破解主线（同时保留"查看源码找线索"的正向玩法）。

- 伪造的后台截图 / 文档底图（含可涂黑区域）。

## 8. 密钥与安全

- 图片生成 API key 存放于 `.env`，已在 `.gitignore` 忽略，绝不写入源码或本文档。
- 提醒：该 key 曾在对话中明文出现，项目完成后建议在服务商后台轮换。

## 9. 明确不做（YAGNI）

- 不做账号系统 / 真实用户数据库。
- 不做真实邮件自动回信、真实二维码等跨媒介 ARG 运营（保留为未来扩展）。
- 不做前端框架 SPA。
- 不做多人 / 联机。

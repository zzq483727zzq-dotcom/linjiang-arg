# 临江 · 幸福家园 ARG 重设计规格

**日期：** 2026-07-22  
**状态：** 设计已讨论定稿，待实现计划  
**性质：** 对现有「静园 / serene」ARG 的叙事与站点结构重做（非小修）

---

## 0. 一句话

好友林屿失联后，微信仍在回复、旁人说他「变好了」。  
玩家从手机进入临江市政府站与幸福家园（心理/睡眠中心），经政府新闻评论进入贴吧与弘科生物薄站，再经已故官员进入政府 OA，最终在幸福家园员工端确认：

> **真身已被处理；社会面运行的是替代方案产物；上游技术方是弘科生物；黑心目的是骗取试点与补贴。**  
> 玩家被系统记录，终局面对警告页与红字墙：「我们会找到你的！」

## 1. 世界与主核

### 1.1 恐怖调性

| 要 | 不要 |
|----|------|
| 正规网页反差、细思极恐 | 民俗、香火、邪神、祈福祭祀 UI（晦气，否决） |
| 后半界面邪 + 临床/产业流水线邪 | 开膛说明书、直球怪物跳脸 |
| 「来晚了」——真身已处理 | 主线拯救真身 |
| 号还在、旁人认领、文件说已处理 | 空泛「壳/内核」说明书词 |
| 被监视感（演出，非真采集） | 真开摄像头、真收集玩家 IP 用于其它目的 |

### 1.2 核心真相

| 层级 | 内容 |
|------|------|
| 事件 | 林屿失踪（对外可曾有寻人，后被结案话术盖住） |
| 真身 | **已处理 / 终止**（不可复活） |
| 社会面 | 替代方案维持的「林屿」：微信代应答、无痣近照、旁人说变好了 |
| 执行方 | **临江市幸福家园**（对外：心理健康 / 睡眠支持） |
| 上游 | **弘科生物**（对外：生物医药 / 转化医学；对内：替代相关管线） |
| 钱 | **骗政府试点经费与补贴**；奇迹康复是假报表道具；林屿是批次耗材而非单独天价定制 |
| 政府 | 公开站维持正面叙事；OA 内可见口径与对接；已故官员曾牵头相关线 |

### 1.3 人物

| 谁 | 角色 |
|----|------|
| **玩家** | 林屿好友/同学；开场说明页给出**姓名**；可被写入关注/下一期评估 |
| **林屿（真）** | 已处理；仅存在于档案、处理记录、失败/终止文书 |
| **林屿（社会面）** | 微信客服式回复；朋友圈过正自拍（无痣）；不正面长见面 |
| **陈默** | 共同好友；口语吐槽与转述「家人说他变好了」；**不**承担贴吧主入口 |
| **家属** | 多经转述出现；认「变好了」 |
| **已故官员** | 政府站讣告 + 遗像；公开信息可拼 OA 账密；死后权限叙事仍冷 |

### 1.4 舞台总览

1. **开头说明页** → 按钮「开始调查」  
2. **手机** — 微信（含朋友圈）· 邮件 · 小红书 · 浏览器 · 电话/短信  
3. **临江市政府公开站** — 少而精新闻 + 强搜索 + **篇篇有评论**  
4. **临江吧**（贴吧感迷你站）— 1～2 帖可点  
5. **弘科生物**薄官网 + **半页看板登录（F3）**  
6. **市政府 OA** — F1 附件（内部聊天/纪要，**不挂贴吧链**）  
7. **幸福家园公开站** — 短、干净  
8. **幸福家园员工端** — 主恐怖场；爆点；监视；终局红字墙前段  

### 1.5 主线节奏（逻辑顺序，可轻微交错）

```
说明页（姓名 + 背景）
  → 手机：林屿不对劲 / 邮件或小红书触达家园或政府
  → 政府站：新闻 + 搜索（寻人/撤稿/试点/弘科两篇）
  → 弘科相关新闻评论区 → 临江吧（F2）
  → 弘科薄站（铺垫）→ 吧内钥匙 → 弘科半页看板（F3）对码
  → 讣告遗像线 → 政府 OA → F1 附件（人话：社会面 / 终止 / 弘科催批次）
  → 幸福家园公开站（可更早穿插）→ 员工端
  → 爆点链 → 监视套件 → 警告页 → 红字墙「我们会找到你的！」→ 结局
```

### 1.6 明确删除或降权（相对旧静园版）

- 博客迷宫主路径、互助论坛主门禁  
- 聊天记录「网页导出链接」  
- 第 48 席 / 香火名册 / 民俗归位 / 胸腔图硬接  
- 莫名档案号密码  
- 静园作为主品牌（由幸福家园承接；旧名可不留）  
- OA 公文直接挂贴吧 URL（已否决，突兀）

## 2. 开头与手机

### 2.1 说明页（最先）

- 调性：案件简报 / 私人备忘，非游戏花活菜单  
- 必备：时间地点（临江）、你与林屿是好友、他失联但有人说没事、**你的名字**  
- 主按钮：**开始调查** → 假手机桌面  
- 不剧透实验、替代、弘科黑幕  

**姓名规则：** 固定默认名或极短输入（实现二选一）；用于名册、审计、红字墙点名。

### 2.2 手机桌面（重做）

- Dock 与桌面图标**不重复**  
- Dock 建议：微信 · 邮件 · 浏览器 · 电话  
- 桌面另放：小红书、短信（可与电话合并入口）  
- 无游戏 HUD；最多微信未读角标  

### 2.3 微信（深度 B + 互动 C）

**结构（真微信）：** 底部 Tab = 微信 | 通讯录 | 发现 | 我  

| 区域 | 内容 |
|------|------|
| 会话 | 置顶林屿；陈默；可选家属向转述号 |
| 林屿聊 | 失联前正常 → 空窗 → 失联后短、稳、像话术 |
| 陈默 | 口语；禁用「公域」等词；转述「变好了」 |
| 发现→朋友圈 | 4～6 条时间线 |
| 通讯录/我 | 薄壳 |

**林屿互动 C：** 历史只读为主；1～2 个节点可发（如问位置、问痣）→ 固定延迟更空回复 → 不再触发。  

**朋友圈：**  
- 失联前：日常；**至少 1 条可见左眼痣**（痣的主证据在此）  
- 失联后：0～2 条过正、**无痣**；评论有人说精神多了  

**不做：** 朋友圈独立 App；聊天导出网页；贴吧主链放微信（避免过早）。

### 2.4 邮件

- 有前因：误发/紧急联系人/回访沾边  
- 主链进**幸福家园**（经浏览器）  
- 像真邮件，不靠上来权限恐吓  

### 2.5 小红书

- **1～2 帖**精做；可提政府试点/家园  
- **不**承担贴吧主入口（避免早于政府站）  
- 配图生活实拍风  

### 2.6 浏览器

- 可解析：政府域、幸福家园域、弘科域、临江吧域  
- 邮件/评论/历史可预填；乱输失败提示像真浏览器  

### 2.7 电话 / 短信（薄）

- 未接、幸福家园勿回复短信、废验证码增真实  
- 不做完整通话系统

## 3. 临江市政府公开站

### 3.1 定位

普通地级市门户：土蓝、可信。  
**主线索库：** 少文 + 强搜索 + **评论区**。  
公开站**不做**腐化特效与鬼图；诡异来自口径裂缝、撤稿痕迹、过亮成功叙事、评论尖声。

### 3.2 信息架构

- 首页：3～5 头条 + 搜索 + 常用入口  
- 新闻：总数约 **6～10 篇**，每篇有职能  
- 通知：2～4 条  
- 政务公开：薄  
- 页脚：办公系统 → OA 登录  
- **每篇可点开新闻均有评论区**  

### 3.3 必含文章职能（含弘科两篇）

| 职能 | 方向 |
|------|------|
| 试点/家园合作 | 幸福家园合法背书 |
| 寻人短讯 | 失踪曾公开 |
| 撤稿/已联系说明 | 与寻人对读发冷 |
| 已故官员讣告 | **遗像** + 生平 + 日期职务；OA 钥匙素材 |
| 官员追记/项目 | 隐晦拼口令 |
| **弘科甲** | 试点技术支持单位弘科生物（正面） |
| **弘科乙** | 产业/转化落地或真实世界研究（产业皮） |
| 弱相关市政 | 真实感填料，评论全正常 |

搜索：`寻人` `幸福家园` `睡眠` `试点` `官员姓名` `弘科` `弘科生物` 等可命中。  
搜官员名 → 讣告+追记。公开站搜玩家名：**无结果或极克制**（有结果留给内网）。

### 3.4 评论区（贴吧主发现位）

- 组件统一；只读或假登录才能发  
- **大多数正常**：捧、骂、跑题、短句，有脑有生活感  
- **少数异常**（弘科稿、寻人/撤稿下）：像真人多嘴  
  - 例方向：「技术支持是不是高新区弘科？亲戚面过 JD 邪门。」  
  - 「临江吧搜批次，看完再给这篇点赞。」→ **贴吧主入口**  
  - 寻人稿下：「已联系？家属群还在问。」  
- 普通市政稿**不**硬塞谜  
- 禁止谜语人套话、禁止官博口吻假网友  

### 3.5 诡异（无鬼图）

- 寻人 vs 撤稿日期/主体不一致  
- 稿件已撤灰链、附件名在打不开  
- 成功叙事过亮  
- 官员线突然断档  

### 3.6 讣告

- 标准遗像一张（严肃证件/会议风，不恐怖化）  
- 套话越规范越冷  
- 口令 100% 可从讣告+追记等 **2～3 处公开信息**拼出

## 4. 幸福家园公开站

### 4.1 定位

- 名：临江市幸福家园  
- 业：心理健康 / 睡眠支持；可挂市级试点合作  
- 林屿最后公开出现处  
- **短站 5～7 页**；干净；邪只在员工端  

### 4.2 页面

首页 · 关于 · 服务 · 案例（1～2 个匿名成功叙事）· 短评估（6～8 题）· 联系 · 页脚工作人员入口  

### 4.3 规则

- 案例可 L 先生式与复制话术同构；不出现真名死亡实验  
- 评估结果永远温和；可写进度状态  
- 员工账密**不**写在公开站；来自 OA 对接  
- 页脚可链政府；可小字技术合作弘科（辅入口）  
- **不做**博客迷宫、论坛入口、聊天导出

## 5. 弘科生物

### 5.1 公开站（薄 · A 档）

- 名：**弘科生物**（全称实现时可写有限公司）  
- 皮：神经/睡眠/精神健康转化医学  
- 约 5 页：首页 / 管线 / 关于 / 合作或新闻 / 联系·招聘  
- **主入口：** 政府新闻点名 + 搜索两篇；浏览器打开  
- **辅：** 家园页脚、OA 合同字段  

**铺垫型泄密（内部感，未钉死）：**  
1. 招聘 JD（材料交接、终止文书等过界人话）  
2. 管线编号（后文与家园档案对码）  
3. 新闻/半涂黑 PDF 一页  
4. 伦理合规反讽页  
5. 图/文件名轻彩蛋（主线不依赖）  

不做：红字墙、摄像头、完整处理林屿记录。

### 5.2 半页看板登录（F3）

- 仅 **1 页**项目看板：批次、合作方幸福家园、状态「社会面运行 / 原对象终止」  
- 账密来自贴吧 F2（测试/试点账号未关）  
- 编号与家园林屿档案对码 = 铁证  
- **防卡死：** 无 F3 也可靠 F1+编号叙述推进到员工端；F3 为铁证加强

## 6. 临江吧（F2 · 贴吧感）

### 6.1 定位

- 气质：贴吧（标题党、楼层、骂战）  
- 体量：**列表可假，真开 1～2 帖**  
- **主入口：** 政府站**弘科相关新闻评论**提吧名/搜法（及寻人稿互证可选）  
- **不**从 OA 附件外链进入（突兀，已否决）  
- **不**从开场微信/小红书塞主链（过早）  

### 6.2 主帖职能

- 前员工/知情者爆料：家园执行、弘科方案、社会面、原对象处理  
- 打码图/工牌/编号  
- **F3 测试账号暗示**  
- 回帖：有人骂谣、有人附和——像真贴吧

## 7. 政府 OA（F1）

### 7.1 进入

公开站页脚办公系统 → 已故官员账密（公开可拼）。

### 7.2 模块

待办 · 公文 · 项目库 · **对接单（家园+弘科）** · 通讯录（官员停用） · 搜索  

### 7.3 F1 附件

- 工作群纪要 / 邮件导出 / 舆情材料（体制内合理）  
- 人话示例方向：「社会面版本先上」「原来的按终止走」「弘科又催批次」  
- **可**提「网上别信临江吧」四字互证  
- **不**直接贴贴吧 URL  

### 7.4 必交给玩家

1. 寻人对外口径被协调  
2. 家园执行、弘科技术支持  
3. 员工端登录线索（对接单工号/临时口令）  

调性：真 OA；越深扫描件/红章可增多；不做法术风。

## 8. 幸福家园员工端（主恐怖）

### 8.1 进入

OA 对接材料 → 公开站工作人员登录。

### 8.2 层深与爆点

| 层 | 内容 | 爆点 |
|----|------|------|
| L0 | 普通后台皮 | — |
| L1 | 林屿档案「已回归/社会面正常」 | T4 假正常 |
| L2 | 版本/对照/项目（人话：原来的 / 外面用的那个） | 缝 |
| L3 | **处理记录 + 克制图（空床/遮盖/静帧）** | **爆点1 T5=B** |
| 同时 | 微信社会面新气泡或可发节点空回；日志亲友关注 | **爆点2** |
| 案例墙 | 无痣林屿；社会面运行中 | **爆点3** |
| 名册 | 玩家姓名在「建议纳入/优先评估」；来源=检索对象好友 | **爆点4 T7 必经** |

**术语人话：** 已处理、原来的、外面用的那个、已通知家属情况稳定；少用 discarded/replica 等灰词当主文案。

### 8.3 监视套件（演出）

| 手段 | 位置 |
|------|------|
| 顶栏访问/在线人数 | 员工端；加深时人数异常 |
| 登录/操作/IP 已记录 | OA 轻；员工端重 |
| **摄像头权限** | **时机 B**：案例墙或「你在表上」；**左上角**仿浏览器权限气泡 |
| 允许/拒绝 | 皆不真开摄像头；皆「已记录…」 |

### 8.4 终局刀 X+Y

- **X：** 警告前微信社会面：「别查了，对你不好。」（知其在查）  
- **Y：** 被监视感收束；预览格可出现终端示意（插画级，非真桌面偷窥）  
- **红字墙：** 系统警告（IP/访问已记录）→ 全屏黑底大红重复字，主句 **「我们会找到你的！」**；可夹玩家名、假会话号  
- 触发：高进度 `flagged` 后，不随机早爆  

### 8.5 林屿形象

- **全套 AI 重做**（旧图淘汰）  
- 同一张脸：有痣（旧）/ 无痣（社会面）  
- 路人感 27 岁东亚男性；非网红非阴间修图  

### 8.6 AI 生图方向（员工端）

空床、夜班走廊、统一微笑档案照、监控静帧、涂黑件、红章、出库/处理单特写。  
禁止：民俗祭坛、开膛、清晰怪物。

## 9. 硬证据闭环（F1+F2+F3）

```
政府新闻/搜索弘科两篇 + 评论区
  → 临江吧 F2 + 弘科薄站铺垫
  → 吧内 → F3 半页看板（对码）
  → 讣告 → OA → F1 附件（人话钉死上下游）
  → 家园员工端（执行与真身与玩家）
```

玩家应能自行说全：

> 假人方案来自弘科；幸福家园执行；真林屿已处理；政府用试点成绩盖住；我被看见了。

## 10. 进度、谜题、结局

### 10.1 阶段（只升不降，示意）

0 说明页姓名 → 1 社交已读 → 2 家园评估/访问 → 3 讣告/弘科关键公开信息 → 4 OA → 5 员工端 → 6 真身处理已见 → 7 flagged 监视完成 → 8 结局  

主门禁钥匙：**OA**、**家园员工端**；F3 为铁证加强。

### 10.2 谜题原则

| 门 | 线索位置 |
|----|----------|
| OA | 讣告遗像页 + 追记等公开 2～3 处 |
| 员工端 | 仅 OA 对接材料 |
| F3 | 贴吧帖内 |

失败分级提示；禁止钥匙只在 README；保留 dev 软开通道可选。

### 10.3 状态字段（示意）

`playerName, stage, socialRead, assessmentDone, obitFound, hongkeFound, oaAccess, staffAccess, truthSeen, flagged, ending, chatNodeUsed, visitFlags[]`  

废弃：rite / 香火 / 48 席等旧字段。

### 10.4 结局（真身不可救）
